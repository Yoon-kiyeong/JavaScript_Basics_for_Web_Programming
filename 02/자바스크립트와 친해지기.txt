자바스크립트 소스를 작성할 때 지켜야 할 규칙
규칙 1 - 대소문자를 구별하여 소스를 작성한다.
자바스크립트는 대소문자를 구별한다.

규칙 2 - 읽기 쉽게 들여 쓰는 습관을 들인다.
자바스크립트 해석기는 소스를 처리할 때 들여쓰기를 신경 쓰지 않는다.
하지만 우리가 작성한 소스를 다른 프로그래머가 읽거나, 오류가 발생해 소스를 수정해야 할 수도 있다.

규칙 3 - 세미콜론으로 문장을 구분한다.
자바스크립트에서 세미콜론(;)은 문장 끝을 나타낸다. 하지만 자바스크립트는 문장 끝에 세미콜론을 붙이지 않아도 잘 실행된다.
실수로 줄 바꿈을 하지 않는다면 오류가 발생한다. 이를 방지하기 위해 자바스크립트 문장의 끝에는 반드시 세미콜론을 붙이는 것이 좋다.

규칙 4 - 자바스크립트 소스에 메모를 하려면 주석을 사용한다.
자바스크립트 소스를 다른 사람에게 설명하고 싶거나, 나중을 위해 안내표시를 하고 싶다면 주석을 사용하면 된다.
주석은 두 가지 방법으로 사용할 수 있다.

한 줄 주석 기본형
// 한 줄 주석은 이렇게 입력한다.

여러 줄 주석 기본형
/* */ 자바스크립트에서 여러 줄의 주석을 표시할 수도 있다.

규칙 5 - 식별자는 정해진 규칙을 지켜 작성한다.
식별자(identifier)는 자바스크립트 문법의 핵심 요소인 변수, 함수, 속성 들을 구별하기 위해 이름 붙여 준 특정 단어를 의미한다.
식별자는 원하는 대로 작성하면 된다. 단, 식별자의 첫 번째 글자는 문자, 밑줄(_), 달러 기호($)로 시작해야 하낟.

규칙 6 - 예약어는 식별자로 사용할 수 없다.
예약어(Keyword)는 자바 스크립트에 먼저 등록된 요소를 가리킨다.
즉, 예약어는 식별자로 사용할 수 없다.
예를 들어 변수를 선언할 때 사용하는 var 예약어는 식별자로 사용하면 오류가 발생한다.

<script>태그는 HTML 문서의 어디에든 삽입한다.
<script>태그는 한 문서 안에서 여러 번 사용할 수 있다.
HTML 문서에 연결하는 외부 자바스크립트 파일의 확장자는 js이다.
간단한 자바스크립트는 크롬 브라우저의 콘솔 창에서 실행하고 결과를 확인할 수 있다.
사용자에게 간단한 텍스트 입력값을 받기 위해 prompt 함수를 사용한다.
사용자에게 알림 창을 표시할 때는 alert 함수를 사용한다.
웹 브라우저 창에 내용을 출력하는 프로그램은 document.write 함수를 사용한다.
자바스크립트 소스에서는 대소문자를 구별한다.
주석은 자바스크립트 소스에 설명을 남겨 두기 위한 것이다.
예약어는 식별자로 사용할 수 없다.
